# 浏览器书签 WebDAV 加密备份插件

一款轻量级浏览器扩展，支持将书签备份到 WebDAV 服务器，并提供 AES-256-GCM 加密、双向同步、版本轮转等功能。

## 功能特性

- **WebDAV 备份**：将书签导出并上传至任意 WebDAV 服务器
- **版本轮转**：自动保留 2 份备份（最新 v1 + 历史 v2），旧备份自动覆盖
- **加密存储**：可选 AES-256-GCM 加密，PBKDF2 密钥派生，防篡改防泄露
- **双向同步**：支持从 WebDAV 拉取书签合并到本地，已存在则跳过，不存在则创建
- **版本选择**：恢复时可自由选择从 v1（最新）或 v2（历史）恢复
- **右键菜单**：页面右键即可快速备份/恢复/打开设置
- **跨浏览器**：兼容 Chrome、Edge、Firefox（Manifest V3）

## 安装方式

### Chrome / Edge

1. 下载本项目源码
2. 打开浏览器扩展管理页：
   - Chrome：`chrome://extensions/`
   - Edge：`edge://extensions/`
3. 开启右上角的「开发者模式」
4. 点击「加载已解压的扩展程序」，选择本项目目录
5. 安装完成，点击工具栏扩展图标即可使用

### Firefox

1. 打开 `about:debugging#/runtime/this-firefox`
2. 点击「临时加载附加组件」
3. 选择项目中的 `manifest.json` 文件
4. 安装完成

## 使用指南

### 1. 配置 WebDAV

点击扩展图标打开设置面板，填写：

| 配置项       | 说明                                                |
| --------- | ------------------------------------------------- |
| WebDAV 地址 | WebDAV 服务器地址，如 `https://dav.example.com/dav/user` |
| 用户名       | WebDAV 认证用户名                                      |
| 密码        | WebDAV 认证密码                                       |
| 备份目录      | WebDAV 上的备份目录路径，默认 `/bookmark-backup`             |

> WebDAV 服务网络上有很多免费提供商，可自行寻找。常见选择包括：
>
> - 坚果云（国内推荐）
> - Nextcloud / ownCloud（自建）
> - Alist（聚合网盘）
> - 各类支持 WebDAV 的云存储服务

### 2. 加密设置（可选）

在设置面板中勾选「启用加密」并设置加密密码。启用后，备份数据将以 AES-256-GCM 加密存储到 WebDAV，恢复时需使用相同密码解密。

> 请牢记加密密码，丢失后将无法恢复备份数据。

### 3. 备份书签

- **方式一**：点击扩展图标 → 「备份到 WebDAV」
- **方式二**：在任意页面右键 → 「书签备份」→ 「备份到 WebDAV」

备份完成后，WebDAV 服务器上会生成 `bookmarks_v1.json` 文件。

### 4. 恢复书签

- **方式一**：点击扩展图标 → 选择恢复版本（最新 v1 / 历史 v2）→ 「从 WebDAV 恢复」
- **方式二**：在任意页面右键 → 「书签备份」→ 选择对应版本恢复

恢复采用**合并模式**：本地已存在的书签（按 URL 判断）会跳过，不存在的书签和文件夹会自动创建，不会覆盖或丢失本地数据。

### 5. 测试连接

点击「测试连接」按钮，验证 WebDAV 配置是否正确、网络是否可达。

## 右键菜单

在任意页面右键，可看到「书签备份」子菜单：

```
📚 书签备份
  ├── ⬆️ 备份到WebDAV
  ├── ⬇️ 恢复最新备份(v1)
  ├── ⬇️ 恢复历史备份(v2)
  ├── ─────────────
  └── ⚙️ 设置
```

操作结果会通过系统通知（Notification）反馈。

### 加密方案

| 组件   | 实现                                           |
| ---- | -------------------------------------------- |
| 加密算法 | AES-256-GCM（认证加密，防篡改）                        |
| 密钥派生 | PBKDF2（SHA-256，100,000 次迭代）                  |
| Salt | 每次加密随机生成 16 字节                               |
| IV   | 每次加密随机生成 12 字节（NIST 推荐）                      |
| 存储格式 | Base64(salt\[16] + iv\[12] + 密文 + 认证标签\[16]) |
| 加密标识 | `ENC1:` 前缀，恢复时可靠判断数据是否加密                     |

### 备份策略

采用**轮转槽位策略**，始终保留 2 份备份：

```
备份时：
  1. 如果 v1 存在，将 v1 移动到 v2（覆盖旧 v2）
  2. 将当前书签导出并上传为 v1

恢复时：
  用户可选择从 v1（最新）或 v2（历史）恢复
```

## 权限说明

| --------------- | ----------------- |
| `bookmarks`     | 读取和创建书签           |
| `storage`       | 保存 WebDAV 配置和加密密码 |
| `contextMenus`  | 注册右键菜单            |

| `<all_urls>`    | 访问任意 WebDAV 服务器地址 |

## 注意事项

1. **CORS 问题**：部分 WebDAV 服务器未配置跨域资源共享（CORS），可能导致浏览器请求被拒绝。请确保服务器允许跨域访问。
2. **密码安全**：WebDAV 密码和加密密码存储在浏览器本地存储中，建议在个人设备上使用。
3. **书签去重**：恢复时以 URL 为唯一标识进行去重，同一 URL 的不同标题书签会被跳过。
4. **大书签文件**：书签数量极大时（>5000），备份和恢复可能需要较长时间，请耐心等待。
